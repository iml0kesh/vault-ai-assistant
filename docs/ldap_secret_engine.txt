VAULT LDAP SECRET ENGINE - COMPLETE GUIDE

PURPOSE
The LDAP Secret Engine allows Vault to manage LDAP credentials and service account
passwords. Vault can dynamically rotate LDAP passwords and provide applications
with current credentials without manual intervention.

WHEN TO USE LDAP SECRET ENGINE
- Application needs to authenticate to an LDAP directory (Active Directory, OpenLDAP)
- Service account passwords for LDAP need to be rotated automatically
- Application needs to bind to LDAP for user lookups
- Enterprise applications connecting to Active Directory

WHEN NOT TO USE LDAP SECRET ENGINE
- For Oracle DB credentials (use Oracle Database Secret Engine instead)
- For static API keys (use KV Secret Engine instead)
- For general application passwords not related to LDAP (use KV Secret Engine)

HOW LDAP SECRET ENGINE WORKS
1. Vault connects to LDAP/Active Directory with a privileged bind account
2. Vault manages service account passwords in LDAP
3. Application requests LDAP credentials from Vault
4. Vault returns current username and rotated password
5. Vault automatically rotates the password on a schedule or on-demand

LDAP SECRET ENGINE ONBOARDING WORKFLOW

Step 1: Enable LDAP Secret Engine
Command: vault secrets enable ldap
Note: Only needed once per Vault cluster.

Step 2: Configure LDAP Connection
Command:
vault write ldap/config \
  binddn="CN=vault-bind,OU=ServiceAccounts,DC=company,DC=com" \
  bindpass="vaultbindpassword" \
  url="ldaps://ldap.company.com:636" \
  userdn="OU=ServiceAccounts,DC=company,DC=com"

Parameters:
- binddn: DN of the Vault privileged bind account in LDAP
- bindpass: Password for the bind account
- url: LDAP server URL (use ldaps for SSL)
- userdn: Base DN for user lookups

Step 3: Create a Static Role (for service accounts)
A static role manages a specific LDAP service account and rotates its password.
Command:
vault write ldap/static-role/payment-ldap-role \
  dn="CN=payment-svc,OU=ServiceAccounts,DC=company,DC=com" \
  username="payment-svc" \
  rotation_period=24h

Parameters:
- dn: Full Distinguished Name of the service account in LDAP
- username: LDAP username of the service account
- rotation_period: How often Vault rotates the password automatically

Step 4: Create Vault Policy
path "ldap/static-cred/payment-ldap-role" {
  capabilities = ["read"]
}
Command: vault policy write payment-ldap-policy payment-ldap-policy.hcl

Step 5: Attach Policy to Auth Method
Attach this policy to the AppRole or Kubernetes role of the application.

Step 6: Application Requests LDAP Credentials
Command: vault read ldap/static-cred/payment-ldap-role
Output:
  username = payment-svc
  password = RotatedSecurePassword123!
  last_vault_rotation = 2024-01-15T10:00:00Z
  rotation_period = 24h

Step 7: Application Uses LDAP Credentials
Application uses returned username and password to bind to LDAP.

REQUIRED INPUTS FOR LDAP SECRET ENGINE ONBOARDING
- Application Name
- LDAP Server URL
- Service Account DN (Distinguished Name)
- Service Account Username
- Vault privileged bind account DN and password
- Password rotation period preference
- Environment (dev / qa / prod)
- Vault Namespace

LDAP SECRET ENGINE POLICY EXAMPLES

Read static credentials:
path "ldap/static-cred/payment-ldap-role" {
  capabilities = ["read"]
}

Rotate credentials on-demand (admin use):
path "ldap/rotate-role/payment-ldap-role" {
  capabilities = ["update"]
}

LDAP SECRET ENGINE BEST PRACTICES
- Use a dedicated privileged bind account for Vault (not a shared admin account)
- Use LDAPS (LDAP over SSL) not plain LDAP for connection
- Set rotation_period based on security policy (24h or shorter recommended)
- Rotate the Vault bind account password regularly
- Monitor LDAP audit logs for Vault-managed credential usage
- Do not share LDAP static roles between applications
- Separate roles per environment (dev/qa/prod)
- Test rotation in lower environments before production
- Ensure application can handle credential rotation (re-auth when password changes)

MANUAL ROTATION
Trigger immediate password rotation:
Command: vault write -f ldap/rotate-role/payment-ldap-role

TROUBLESHOOTING LDAP SECRET ENGINE

Error: LDAP connection failed
Cause: Wrong LDAP URL or network issue
Fix: Verify URL and connectivity from Vault server to LDAP server

Error: Bind failed - invalid credentials
Cause: Wrong binddn or bindpass
Fix: Verify bind account credentials

Error: Service account not found
Cause: Wrong DN in static role
Fix: Verify DN exists in LDAP directory

Error: Permission denied reading ldap/static-cred
Cause: Policy path wrong or not attached
Fix: Verify policy path is "ldap/static-cred/<role-name>"

Error: Password rotation failed
Cause: Vault bind account lacks permission to change passwords in LDAP
Fix: Grant password reset permission to Vault's bind account in Active Directory
